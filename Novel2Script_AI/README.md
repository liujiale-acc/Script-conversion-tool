AI 影视剧本全链生产系统 (Script-Expert 2.1)

1. 项目愿景
Script-Expert 2.1 (Industrial Edition) 是一款面向影视工业化的智能辅助创作工具。在 v2.0 实现“剧本医生”与“人机协作”的基础上，v2.1 进一步引入了全量数字化映射技术。它不仅能生成供人类阅读的艺术化剧本，更能同步产出符合机器逻辑的 YAML 结构化数据，真正实现了从文学创作到数字制片的无缝链接。

2. 核心功能亮点 (v2.1 深度进化)

- 【硬核】YAML 全量数字化映射：
  - 1:1 结构化匹配：完美响应“议题三”要求，将剧本中的场景、动作、对白、神态等文学元素，精准转化为高精度的 YAML 数据流。
  - 工业级接口：生成的 YAML 可直接驱动 3D 渲染引擎或制片管理软件。
- 【核心】AI 剧本医生 (Script Doctor)：
  - 量化诊断看板：实时提取 AI 对剧本的张力评分、节奏建议与冲突分析，以黄色看板形式独立展示。
- 【突破】长文本重构能力：
  - 多章节处理：针对 3 个章节以上的小说文本进行了逻辑优化，确保长篇叙事在转换过程中的角色稳定性。
- 【灵活】双轨交互与导出：
  - 协作编辑器：预览区支持实时手动润色。
  - 双格式输出：支持一键导出专业 Word 文档（给人看）与标准 YAML 源码（给机器看/用于交卷）。
- 【视觉】分镜提示词生成**：
  - 自动为每个场景生成适配 Midjourney/SD 的 AI 绘图提示词，辅助导演构建画面感。

3. 对“挑战赛-议题三”的要求适配
本项目严格按照挑战赛官方要求进行开发，达成以下硬性指标：
1.  自动转换：实现了从小说原文到结构化 YAML 格式的自动化生成。
2.  长文本支持：系统架构支持处理 3 个章节以上的小说录入。
3.  Schema 文档：仓库中额外附带了 `YAML_SCHEMA_DEFINITION.md`，详细定义了协议设计初衷。

4. 技术架构
- 大模型引擎：DeepSeek-V3 (基于 OpenAI SDK 兼容接口)。
- 后端架构：Python 3.9 + Flask，实现了多板块数据标记与切分逻辑。
- 文档自动化：`python-docx` 驱动的高保真渲染引擎。
- 前端交互：基于 Tab-View 的双轨预览界面，支持实时异步数据同步。

5. 演进历程
- v1.0 (基础版): 完成 Flask 架构，实现小说到剧本的基础转换。
- v2.0 (专业版): 引入“剧本医生”评分看板与可编辑工作台。
- v2.1 (工业版): 
    - 重大更新：上线 YAML 结构化映射引擎，达成议题硬性指标。
    - 稳定性修复：通过唯一标签切分法，彻底解决多板块输出下的数据解析问题。

6. 环境配置
1. 克隆仓库：`git clone [您的仓库地址]`
2. 安装依赖：`pip install -r requirements.txt`
3. 配置环境：在 `.env` 中填入 `DEEPSEEK_API_KEY`。

7. 启动应用
python app.py
