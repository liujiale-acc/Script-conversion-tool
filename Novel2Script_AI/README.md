AI 影视剧本全链生产系统 (Script-Expert 3.0)

1. 项目愿景
Script-Expert 3.0 (Production Edition)** 是一款面向影视工业全链路的智能生产引擎。在 v2.1 数字化映射的基础上，v3.0 实现了从“文学创作”到“生产管理”的重大跨越。系统不仅能生成高质量剧本与结构化 YAML，更深度激活了数据价值，上线了副导演自动化制片拆解模块，实现了剧本元素到制片通告单的无缝转换，为影视剧组提供真正的工业级降本增效方案。

2. 核心功能亮点 (v3.0 生产级演进)

- 【制片级】自动化副导演拆表 (The AD Breakdown)：
  - 全要素自动统计：基于 YAML 数据流，系统可自动计算总场次、主要拍摄地、内外景比例，并一键生成包含场景、人物、道具、日夜景的专业制片看板。
  - 关键道具预警：AI 自动识别剧本正文中的核心道具并进行归类统计，辅助美术组与道具组提前介入生产逻辑。
- 【数字化】YAML 全量数字化映射 (Topic 3 Compliance)：
  - 1:1 结构化匹配：完美响应“议题三”要求，将剧本中的动作、对白、神态等元素，精准转化为高精度的 YAML 数据，直接对接 3D 渲染引擎或数字统筹软件。
- 【多规格】全品类适配引擎：
  - 差异化创作逻辑：支持电影标准、短剧 (高节奏)、电视剧、动漫/番剧、短视频广告五大规格。针对不同规格，AI 会自动调整叙事节奏、反转频率及制片颗粒度。
- 【智能版】剧本医生诊断看板 (Script Doctor)：
  - 量化诊断看板：实时提取 AI 对剧本的张力评分、节奏建议与冲突分析，以独立看板形式展示，为内容质量提供客观参考。
- 【工业化】双轨交互与高精度导出：
  - 双格式输出：支持导出符合行业标准的专业Word 剧本（包含中括号自动加粗排版）与标准YAML制片源码。
  - 精准解析引擎：前端引入 `js-yaml` 高精度解析库，确保制片数据 100% 还原，拒绝数据错位。

3. 对“挑战赛-议题三”的要求适配
本项目严格按照挑战赛官方要求进行深度开发，圆满达成以下指标：
（1）自动转换：实现了从小说原文到“剧本+YAML+制片报表”的自动化全量生产。
（2） 长文本支持：系统架构深度优化，稳定支持 3 个章节以上的小说录入与结构化拆解。
（3）Schema 文档：仓库附带 `YAML_SCHEMA_DEFINITION.md`，详细定义了数字化映射的工业协议准则。

4. 技术架构
- 大模型引擎：DeepSeek-V3 (基于 OpenAI SDK 兼容接口，采用多级标签切分 Prompt 技术)。
- 后端架构：Python 3.9 + Flask，实现了多板块数据标记切分与文件自动化渲染逻辑。
- 解析引擎：集成 `js-yaml` 库，实现前端数据对象化的精准报表渲染。
- 文档自动化：`python-docx` 驱动的高保真排版引擎，支持动态样式注入。

5. 演进历程
- v1.0 (基础版): 完成 Flask 架构搭建，实现小说到剧本的基础文学转换。
- v2.0 (专业版): 引入“剧本医生”评分看板与双轨协作工作台。
- v2.1 (工业版): 上线 YAML 数字化映射引擎，解决了多板块数据解析的稳定性问题。
- v3.0 (生产版): 
    - 重大突破：上线**副导演自动制片拆表模块**，实现剧本到生产数据的闭环。
    - 功能完善：补全电影、动漫、短剧等多规格创作模式，引入专业级 YAML 解析引擎。

6. 环境配置
1. 克隆仓库：`git clone [您的仓库地址]`
2. 安装依赖：`pip install -r requirements.txt`
3. 配置环境：在项目根目录 `.env` 文件中填入 `DEEPSEEK_API_KEY`。

7. 启动应用
python app.py
