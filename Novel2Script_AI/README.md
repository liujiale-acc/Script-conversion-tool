影视剧本全链生产系统 (Script-Expert 4.0)

1. 项目愿景 (Project Vision)
Script-Expert 4.0是一款面向影视工业全链路的数字化智能生产引擎。本项目旨在打破影视创作中“文学感性描述”与“工业理性制片”之间的信息壁垒，实现从小说原著到生产制片数据的全量自动化转换。通过自研的叙事弧线物理对齐技术与数字化生产要素重构引擎，系统可一键生成专业剧本、高精度数据映射文件（YAML）及数字化制片看板，为现代影视剧组提供工业级的降本增效解决方案。

2. 核心功能亮点 (Key Features)

- 【制片级】全要素智能拆解 (Automated Production Insights)
  - 工业看板生成：基于后端解析逻辑，系统自动提取并展示总场次、独立场景数、内外景占比分布等关键生产指标。
  - 生产要素识别：智能识别原著中的登场人物与关键生产道具，辅助剧组美术、道具与统筹部门提前介入生产准备。

- 【数字化】YAML 结构化生产映射 (Structural Metadata Mapping)
  - 高精度对齐：精准响应数字化制片需求，将剧本中复杂的动作描写、角色对白、环境神态等元素，转化为标准 YAML 数字化源码，直接对接数字统筹系统。

- 【多规格】全品类制片适配
  - 差异化逻辑引擎：内置电影标准模式、高频率反转短剧模式、长篇剧集模式、动漫分镜模式及宣传片广告模式，自动调整叙事张力频率与制片拆解颗粒度。

- 【量化级】智能诊断报告 (Smart Diagnostic Report)
  - 叙事张力波形：首创物理阶段分布算法，确保叙事张力曲线与剧作法结构（序幕、发展、冲突、高潮、结局）在可视化轴线上精准映射，量化评估内容节奏。
  - 创作改进建议：同步输出针对长文本叙事的冲突强度分析与节奏优化建议，辅助内容评审与决策。

- 【工业化】标准交付与高保真导出
  - 高保真 Word 导出：支持导出符合行业排版标准的专业 Word 剧本，自动执行场次标签识别与重点排版加粗，实现“生成即开机”。

3. 技术架构 (Technical Architecture)

- 大模型驱动：基于 DeepSeek-V3 引擎，采用多级标签数据切分技术（Tag-Splitting Prompting）。
- 后端框架：Python 3.9 + Flask，实现业务逻辑调度与多板块数据解析。
- 解析引擎：集成 `js-yaml` 高性能解析库，实现前端数据对象化的精准看板渲染。
- 自动化排版：基于 `python-docx` 驱动的高保真样式注入排版引擎。

4. 第三方库与依赖说明 (Dependencies)

本项目运行需安装以下核心依赖库：
- `Flask`: 轻量级后端 Web 框架，处理数据请求与路由分发。
- `openai`: 用于对接大模型推理接口（OpenAI 兼容协议）。
- `python-docx`: 用于生成并处理符合影视行业格式的 Word 文档。
- `python-dotenv`: 用于安全管理 API 密钥等环境变量。
- `js-yaml` (CDN): 前端高性能 YAML 数字化映射解析引擎。

5. 环境配置 (Configuration)
  (1) 克隆仓库：`git clone [您的仓库地址]`
  (2)安装依赖：`pip install flask openai python-docx python-dotenv`
  (3)配置环境：创建 `.env` 文件填入你的 `DEEPSEEK_API_KEY`。
6. 启动应用
python app.py
