# AI 影视剧本全链生产系统 (Script-Expert 2.0)

1. 项目愿景
在 1.0 版本实现“全链条输出”的基础上，**v2.0 致力于打造“人机协同的智能编剧工作台”**。通过引入 AI 剧本诊断算法与实时交互编辑逻辑，本项目实现了从“AI 孤立生成”到“人机深度协作”的跨越，旨在为创作者提供具备专业审美建议与高度定制能力的生产力环境。

2. 核心功能亮点 (v2.0 深度进化)
- 【炸裂】AI 剧本医生 (Script Doctor)：
  - 多维诊断：在生成剧本的同时，实时对剧情的“戏剧张力”、“节奏感”及“核心冲突”进行深度分析并打分。
  - 修改对策：针对 AI 诊断出的逻辑弱点，提供专业编剧视角下的修改建议。
- 【突破】人机协作工作台**：
  - 即时微调：打破“只读预览”限制，生成的剧本预览区升级为可编辑状态，允许编剧在 AI 草稿基础上进行即时润色与二次创作。
- 【闭环】所见即所得 (WYSIWYG) 导出**：
  - 动态捕捉**：升级 Word 导出逻辑，系统将优先捕获编剧在预览区修改后的**最终版本**进行文档渲染，确保导出成果与人工调整实时同步。
- 【优化】智能看板交互**：
  - 诊断反馈窗**：新增独立的“剧本诊断报告”动态看板，实现诊断信息与剧本正文的解耦展示，提升创作时的视觉效率。

3. 技术架构 (升级版)
- 大模型引擎：DeepSeek-V3 (采用更高维度的结构化 System Prompt 策略)。
- 后端架构：Python 3.9 + Flask，新增了基于 POST 链路的动态内容捕获机制。
- 数据流控制：实现了从“模型渲染 -> 前端交互 -> 用户二次编辑 -> 后端重构导出”的完整闭环。
- 文档自动化：`python-docx` 逻辑重构，支持对用户编辑后的文本进行智能识别与专业分段排版。

4. 演进历程 (PR 提交轨迹证明)
本项目严格遵循议题规范进行分步开发，通过 PR 记录展现完整的迭代路径：
- Phase 1 (初稿): 完成 Flask 基础架构搭建，实现小说到剧本的基础格式转换。
- Phase 2 (v1.0): 并行生成角色画像与分镜提示词，确立工业标准排版，解决格式清洗问题。
- Phase 3 (v2.0 现阶段): 
    - 功能跃迁：开发“剧本医生”诊断模块，赋予系统审美评价与逻辑反馈能力。
    - 交互革命：实现预览区全量可编辑化，构建“AI 生成+人工润色”的协同流。
    - 链路闭环：打通前端实时编辑内容与后端 Word 导出引擎的同步链路。

5. 环境配置
1. 克隆仓库：`git clone [您的仓库地址]`
2. 安装依赖：`pip install -r requirements.txt` (含 flask, openai, python-docx, python-dotenv)
3. 配置环境变量：在 `.env` 文件中填入 `DEEPSEEK_API_KEY=您的密钥`

6. 启动应用
python app.py
